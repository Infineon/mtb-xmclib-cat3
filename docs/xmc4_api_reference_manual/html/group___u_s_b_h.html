Detailed Description

The USBH is the host mode device driver for the USB0 hardware module on XMC4000 family of microcontrollers. The USB0 module can be used to establish a USB interface between outside world and XMC4000 family of controllers. The USB module includes the following features in host mode:

  1. Complies with the USB 2.0 Specification.
  2. Supports up to 14 bidirectional pipes, including control pipe 0.
  3. Supports SOFs in Full-Speed modes.
  4. Supports clock gating for power saving.
  5. Supports FIFO mode data transaction.

The below figure shows the overview of USB0 module in XMC4 microntroller.

The USBH device driver supports the following features:

  1. Initialize/Uninitialize the USB0 module on XMC4000 device.
  2. Control VBUS state.
  3. Reset USB port.
  4. Set the USB device address.
  5. Allocate pipe for new endpoint communication.
  6. Modify an existing pipe.
  7. Transfer data on selected pipe.
  8. Abort ongoing data transaction.
  9. Handle multi packet data transaction by updating toggle information.

The USBH device driver expects registration of callback functions XMC_USBH_SignalPortEvent_t and XMC_USBH_SignalPipeEvent_t to be executed when there is port event interrupt and pipe event interrupt respectively.
The USBH driver is CMSIS API compatible. Please use Driver_USBH0 to access the USBH API.
For example, to initialize the USB host controller, use Driver_USBH0.Initialize().

◆ XMC_USBH_HandleIrq()

void XMC_USBH_HandleIrq ( uint32_t  gintsts)
Parameters
gintstsUSB global interrupt status.
Returns
None.
Description:
Updates logical state of the USB host driver based on the input status value. It handles port interrupt and channel interrupt. It responsible for updating data toggle information for multi-packet data transmission. It executes the callback function on transfer completion and reception of data. It also does error management and calls the relevant callback functions to indicate it to the application.
